java中的正则表达式(包教包会!)

无标签
发布日期:   2022-07-17

001 简单介绍

本篇文章主要讲述java中的正则表达式的基本用法，简单精炼，看完都能学会的，与hive sql中正则表达式的用法是相通的

002 语法篇

01 元字符

1、字符匹配符

[] 表示可接收的字符列表 匹配其中的任意一个字符，案例如下：

[abcd]
输出a或者b或者c或者d

[^] 表示不接收的字符列表，案例如下:

[^abc] 
匹配除abc外的任意字符

- 连字符，案例如下：

[A-Z]
匹配任意单个大写字母

. 表示匹配除\n以外的任何字符，案例如下：

a..b
匹配以a开头 b结尾中间包含2个任意字符的长度为4的字符

\\d 表示匹配单个数字字符，[0-9]，案例如下：

\\d&#123;3&#125;(\\d)?
包含3或4个数字的字符串

\\D 匹配单个非数字字符,案例如下:

\\D(\\d)*
以单个非数字字符开头，后面接任意个数数字字符串

\\w 匹配单个数字、大小写字母相当于[0-9a-zA-Z] ，案例如下：

\\d&#123;3&#125;\\w&#123;4&#125;
以3个数字开头的长度为7的数字字母组合

\\W 匹配单个非数字非大小写字母[^0-9a-zA-Z]，案例如下：

\\W+\\d&#123;2&#125;
以至少一个非数字字母开头，2个数字字符结尾的字符串

\\s 表示匹配任何空白字符（空格和制表符等）

\\S 表示匹配任何非空白字符

2、选择匹配符

| 表示匹配｜之前或之后的表达式，案例如下:

ab|dc
ab或dc

3、限定符号

* 指定字符重复0次或者n次，案例如下：

(abc)*
输出abcabcabc

+ 指定字符串重复一次或者n次，案例如下：

m+(abc)*
输出m、mabc

? 指定字符出现0次或者1个，案例如下：

m+abc？
表示以至少1个m开头后面接ab或者abc的字符串 问号限制的c出现的次数

{n} 表示只能输入n个字符，案例如下：

[abcd]&#123;3&#125;
以abcd字母组成的任意长度为3的字符串

{n,} 表示指定至少n个匹配，案例如下:

[abcd]&#123;3,&#125;
由abcd中字母组成的任意长度不小于3的连续字符串

{n,m} 表示指定至少n个但是不多于m个匹配，案例如下：

[abcd]&#123;3,5&#125;
由abcd的字母组成的任意长度不小于3 不大于5的连续字符串

注意⚠️java是默认是贪婪匹配 优先匹配多的
\\d+? 问号写在最后表示非贪婪匹配

4、(?i)

(?i)abc 表示abc不区分大小写
a(?i)bc 表示bc不区分大小写
a((?i)b)c 表示只有b不区分大小写

Pattern.compile(regstr,Pattern.CASE_INSENSITIVE)  当pattern对象指定Pattern.CASE_INSENSITIVE 表示匹配不区分大小写

5、定位符

^ 表示指定起始字符，案例如下：

^[0-9]+[a-z]*
以至少一个数字开头，后面接任意个小写字母的字符串

$ 表示指定结束符，案例如下：

^[0-9]\\-[a-z]+$
以1个数字开头后接连字符“-”
并以至少1个小写字母结尾的字符串

\\b 表示匹配目标字符串的边界，案例如下：

czy\\b
这里说的字符串的边界指的是子串间有空格，或者是目标字符串的结束位置
sasdahan nnczy

\B 表示匹配目标字符串的非边界

czy\B
输出czyssad

6、转义符号\\

需要用到转义符的字符有以下：.*+()$/\?[]^&#123;&#125;

02 常用类

1.Pattern

pattern对象是一个正则表达式对象。pattern类没有公共构造方法。要创建一个pattern对象调用其公共静态方法，它返回一个pattern对象。该方法接收一个正则表达式作为它的第一个参数。

比如：Pattern r=Pattern.compile(pattern);

2.Matcher

Matcher对象是对输入字符串进行解释和匹配的引擎。与Pattern类一样，Matcher也没有公共构造方法。需要调用Pattern对象的matcher方法来获得一个matcher对象
整体匹配验证是否一样
比如qq号验证

String content ="abcdefg";
String regstr="abcdef";
boolean compile=Pattern.matches(regstr,content);
System.out.println("整体匹配="+ complile);

03 分组、捕获、反向引用

1 分组

我们可以用圆括号组成一个比较复杂的匹配模式，那么一个圆括号的部分我们可以看作是一个子表达式/一个分组

1.1 (pattern)

非命名捕获。捕获匹配的字符串编号为零的第一个捕获是由整个正则表达式模式匹配的文本，其他捕获结果则根据左括号的顺序从1开始自动编号

String regstr="(\\d)(\\d)";
matcher.group(0) 得到匹配字符串
matcher.group(1) 得到匹配字符串的第一个分组
matcher.group(2)得到匹配字符串的第二个分组
1.2 (?pattern)

命名捕获。将匹配的子字符串捕获到一个组名称或者编号名称中。用于name的字符串不能包含任何标点符号，并且不能以数字开头。可以使用单引号代替尖括号，例如(?’name’)

String regstr="(?<g1>\\d)(?<g2>\\d)";
matcher.group(0) 得到匹配字符串
matcher.group("g1") 得到匹配字符串的第一个分组
matcher.group("g2")得到匹配字符串的第二个分组
1.3 (?:pattern)

匹配pattern但不捕获该匹配的子表达式，即它是一个非捕获匹配，不存储以后使用的匹配。这个对于用or字符（｜）组合模式部件的情况很有用

原：'industry|industries'
改进:'industr(?:y|ies)'
1.4 (?=pattern)

它是一个非捕获匹配

'windows(?=95|100|2000)'
匹配 windows2000中的windows
但是不匹配windows3.1中的windows
1.5 (?=pattern)

表示匹配不处于pattern的字符串的起始点的搜索字符串

'windows(?=95|100|2000)'
匹配 windows3.1中的windows
但是不匹配windows2000中的windows

2 捕获

把正则表达式中子表达式/分组匹配的内容，保存到内存中以数字编号或显示命名的组里，方便后面引用，从左向右，以分组的左括号为标志，第一个出现的分组的括号为1，第二个为2，以此类推。组0代表的是整个正则式

3 反向引用

圆括号的内容被捕获后，可以在整个括号后被使用，从而写出一个比较实用的匹配模式，这个被我们称为反向引用，这种引用的概念既可以是在正则表达式内部，也可以是正则表达式外部，内部反向引用\\分组号，外部反向引用$分组号

public class regexp01 &#123;
   public static void main(String[] args) &#123;
      String content ="abcdefg11111232 1212 1234567";
      String regstr="(\\d)(\\d)\\1\\2";//==111表示第三位数字引用第一位数字112表示第四位数字引用第二位数字
      Pattern compile = Pattern.compile(regstr);
      Matcher matcher= compile.matcher (content) ;
      while (matcher.find())&#123;
            System.out.println(matcher.group(0));
       &#125;
    &#125;
